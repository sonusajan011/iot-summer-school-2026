# Repository Branching Strategy

- **main**: Production-ready code. Holds the stable, finalized version of the IoT assignments and projects.
- **dev**: Integration branch. Used to combine and test features from specific sub-modules before pushing to main.
- **sensor-module**: Feature branch dedicated to developing sensor interfacing scripts (e.g., DHT11, Ultrasonic sensors).
- **wireless-module**: Feature branch dedicated to communication protocols and networking modules (e.g., Wi-Fi, MQTT, Bluetooth).
